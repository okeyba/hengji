# 进阶 · 多币种

[← 返回手册目录](README.md)

收了外币、囤了点比特币、或者偶尔要换汇——衡记能把这些一起管起来，但不打扰只用人民币的人。

**适用**：有外币收支 / 持有外币或加密资产的小老板 / 进阶用户。日常只进出人民币的话，这一篇可以跳过。

---

## 默认是关的，按需打开

衡记开箱即是纯人民币：账户、报表里都没有「币种」这一栏，少一层干扰。多币种属于商家进阶功能，要分两步打开：

1. 「设置」→ 勾上「**开启商家进阶功能**」。
2. 往下找到「**多币种**」卡片，勾上「开启多币种」。

打开后才会出现币种注册表、建账户时的币种选择、报表里的展示币种切换。只用人民币就一直保持关闭，界面更清爽。

> 关于极简模式和进阶功能开关，详见 [上手与设置](modes.md)。

### 持有外币账户会「锁定为开」

如果你已经建过外币账户（哪怕后来想关），多币种开关会变成**锁定开启、不可取消**——因为「关掉多币种却还显示着外币账户」自相矛盾。这时卡片里会写明是哪些币种在用，例如「已有外币账户在用（美元），多币种已锁定开启」。

要回到纯人民币，先去 [账户](accounts.md) 页把所有外币账户归档，开关才会重新可关。

## 自管币种注册表

衡记不内置一长串币种、也不联网拉汇率——你只登记自己真正用到的几个。这份注册表是**全局共享**的，所有 [账本](books.md) 共用同一套币种。

人民币（CNY，¥）是**本位币**，永远在列表第一行、固定不可改、汇率恒为 1。其余币种由你自己添加，每个填五项：

| 字段 | 含义 | 例子 |
| --- | --- | --- |
| 代码 | 币种代码，会自动转大写 | `JPY`、`USD`、`BTC` |
| 符号 | 显示用符号，留空就用代码 | `¥`、`$`、`₿` |
| 名称 | 中文名，留空就用代码 | 日元、美元、比特币 |
| 小数位 | 最小单位的小数位数（0–8） | 日元填 `0`、比特币填 `8` |
| 汇率 | **1 单位该币种 = 多少人民币** | 美元约 `7.1`、比特币约 `400000` |

添加步骤：

1. 在「多币种」卡片底部那一排输入框里，依次填代码、符号、名称、小数位、汇率。
2. 点「**添加**」。

注意几点：

- **代码不能重复**，也不能是 CNY（那是本位币）。
- **汇率必须大于 0**，否则会提示「请输入有效的对人民币汇率」。
- 已添加的币种，直接点对应单元格改符号 / 名称 / 汇率，**点开输入框外面（失焦）即自动保存**，不用额外按钮。
- 汇率改动只影响**今后报表的折算展示**，不会改写任何已记金额（原币金额永远按原样存）。

### 可变精度：日元 0 位、比特币 8 位

不同币种小数位不一样，衡记按你登记的小数位来记账和显示：

- **日元**填小数位 `0`——1 日元就是最小单位，没有「分」。
- **比特币**填小数位 `8`——一直精确到 0.00000001 BTC（俗称 1 聪）。
- 人民币 / 美元都是 2 位。

> ⚠️ **一旦有账户在用该币种，小数位就锁死改不了。** 因为改小数位会让程序「错读」已经记下的金额。想换精度，得先归档用旧设置的账户。符号、名称、汇率不受此限，随时能改。

### 删除币种

只有**没有任何账户在用**的币种才能删。如果某币种还有账户挂着，删除会被拦下并提示「已有账户在用，不能删除（先归档 / 改用其他币种的账户）」。

## 建账户时选币种

多币种开启后，新建一个真实资金账户时会多出「币种」一项，从你登记过的币种里选。比如：

- 「美元 PayPal」选 USD
- 「币安钱包」选 BTC
- 「门店现金」保持 CNY

账户的币种**决定了它里面每一笔的原币**——往美元账户记的钱按美元算，往人民币账户记的钱按人民币算。账户怎么建、为什么真实账户全局共享，见 [账户](accounts.md)。

## 展示币种：总表折算 + 原币 chip

各个账户的余额、流水里的每一笔，**永远按原币显示**（美元就是 $、日元就是 ¥ 不带小数）。但财务总表要把不同币种放一起看，就需要一个统一的「**展示币种**」来折算。

在「多币种」卡片里有个「展示币种」下拉，从已登记的币种里选一个（默认人民币）。选定后：

- **财务总表、各账本的净资产和收支**，按所选展示币种用汇率折算后显示。
- **各账户余额、每条流水仍按原币**，不动。
- 多币种的资产在总表里会先**按币种分组小计**，每组旁边标着原币（原币 chip），再折合成展示币种汇总。这样既看得到「我有多少美元、多少 CNY」，又有一个折合总数。

折算只是**展示层换算**，底层每笔的原币金额一个都没改。换个展示币种、或改了汇率，看到的折合数字会变，但你实际记的钱分毫不动。

> 原理（点到为止）：每个币种的汇率都登记成「相对人民币」。折算时程序先把原币换成人民币基准，再除以展示币种的汇率换过去。所以你只用维护「对人民币汇率」这一列，展示币种可以随便切。

## 换汇：一笔记成两条原币腿

把人民币换成美元、或卖比特币换回人民币，这类「同一笔交易、两种币种」的操作，衡记当作**账户间转账**来记——只不过转出和转入是不同币种。

做法就是一笔从外币账户 → 另一币种账户（或反过来）的转账：你填转出账户实际扣了多少（按转出账户的原币），再填转入账户实际收了多少（按转入账户的原币）。两条腿各按各自的真实原币金额记，不需要你手算中间汇率。

记好后，这笔在 [流水](recording.md) 里会显示成「**换汇**」（带 💱 图标），副标题写明「从某账户 转出额 → 某账户」，让你一眼看出这是一次跨币种的兑换，而不是普通同币种转账（🔁）。

> 提示：换汇按两腿各自的真实到账金额记，等于把当次的实际成交汇率「钉」在了这笔交易里，跟币种注册表里那个用于报表折算的参考汇率互不影响。
