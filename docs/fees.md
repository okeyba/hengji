# 额外费用与公式引擎

佣金、运费、包装费这类「随单子走、还会因为单子大小变档」的钱，让衡记替你算——开单时按行勾一下，金额自动跳出来，都算进收入。

**适用**：开网店、做社区团购、接平台代发的小生意；订单金额之外还经常加收 / 扣减各种费用的人。

> 这是商家进阶功能。先到「设置 → 开启商家进阶功能」，侧边栏才会出现「额外费用」和「订单」。普通记账用不到，可以略过本篇。

---

## 一句话理解

你先在账本里**定义**好一笔费用（叫什么、怎么算），以后**开单时**在每个商品行勾选它，衡记按你定义的规则实时算出金额，计入这张单的收入。

费用定义是**账本级**的、可复用：在「生意账」账本里建一次「平台佣金」，这个账本以后每张单都能勾。换到「生活账」账本就看不到——费用跟着账本走，和[账本与分账](books.md)是一套逻辑。

---

## 三种计算方式

新增费用时选一种「计算方式」，决定费用怎么从商品行算出来：

| 方式 | 怎么算 | 典型用途 |
|------|--------|----------|
| **百分比** | 勾选行的金额 × 百分数 | 平台佣金（成交额的 4%）、刷卡手续费 |
| **固定金额** | 一次性收这么多，和勾几行无关 | 包装费、起步运费 |
| **按数量** | 勾选行的件数 × 单位额 | 按件算的打包费（每件 ¥1.5） |

注意「固定金额」是**整笔费用一次性**，不是每行各收一次——一张单里这笔费用勾了几行，金额都一样。

---

## 阶梯档位：单子越大，费率自动跳档

每种费用都支持**声明式阶梯**：你列出几档「门槛 → 数值」，衡记看这张单里**勾了这笔费用的那些行合计落在哪一档**，整组就用那一档。

百分比 / 固定金额按**金额合计**定档，按数量按**件数合计**定档。

举例，建一个「平台佣金」，计算方式选百分比，列三档：

| 金额阈值 | 百分比 |
|----------|--------|
| ¥0 | 5% |
| ¥600 | 4% |
| ¥2000 | 3% |

含义：勾了佣金的那些行合计 < ¥600 收 5%，¥600–¥2000 收 4%，≥¥2000 收 3%。一张单勾了 ¥800 的货，整组按 4% 算。

规则上要点：

1. 至少有一档**阈值填 0**（保底档，覆盖最小的合计）。只想要一个固定费率，就只填这一行、阈值留 0。
2. 阈值必须是**非负数**。
3. **数值可以为负**——满减、折扣就靠这个。比如「满 300 减 20」：计算方式选固定金额，列两档，阈值 ¥0 数值 0、阈值 ¥300 数值 −20，合计够 300 就自动减 20。

---

## 第一步：定义一笔费用

侧边栏进「额外费用」：

1. 在「新增费用」里填**名称**（如「佣金」「运费」）。
2. 选**计算方式**（百分比 / 固定金额 / 按数量）。
3. 填阶梯档位。每行一档「阈值 + 数值」，点「＋ 加一档」加更多档，多余的档用「×」删。单档就留阈值 0 那一行。
4. 点「添加」。

列表里每笔费用会显示它的所有档位，例如 `≥¥0 → 5% · ≥¥600 → 4%`。

**编辑 / 归档**：点「编辑」改名字、改方式、改档位。点「归档」让它在开单时不再出现——历史订单里已经记过的费用不受影响，照旧算。归档的费用可以随时「恢复」。

---

## 第二步：开单时勾选

进「订单 → 新建订单」，每个商品行下面有一排「额外费用」按钮（就是你定义过的、未归档的费用）。点一下变高亮表示这一行应用该费用，再点取消。

- 同一笔费用可以勾在**多行**——它们会合并成一组算合计、一起定档。
- 不同行勾不同费用都可以，互不影响。
- 勾选后下方实时显示明细：`商品 ¥800 ＋ 额外费用（佣金 ¥32）`，「合计」一行直接是含费用的总额。

保存订单后，费用就跟这张单绑定了。订单列表里每张单会标「含额外费用：佣金 ¥32」，单子的金额是含费用的总额。

---

## 原理：费用都当收入

完成订单（确认收入）时，衡记把**商品额 + 所有额外费用**一起记进「营业收入」，客户应付、毛利计算也都用这个含费用的总额。背后的复式分录自动生成，你不用管借贷——开单勾费用、完成订单，账就对了。

这意味着：佣金、运费这些在衡记里被看作你这单**收进来的钱**的一部分，而不是单独的支出科目。如果你实际要把佣金付给平台，那是另一笔[流水](recording.md)。

---

## 它是公式引擎的第一块

「额外费用」是衡记规划中的**公式引擎 / 插件地基**落地的第一块原语：一个名字 + 一组声明式阶梯档位，确定性地从订单行算出一个金额。它纯粹、可复算、可审计——同样的输入永远得到同样的结果。

后续更复杂的费用规则、乃至「用大白话描述一条费用规则、自动翻译成档位」都会架在这块原语上。当前版本（v0.3.0）已经能用的就是上面这套：百分比 / 固定 / 按数量 × 阶梯档位。
