# 商品与库存

把进了多少货、手里还剩多少、每件成本多少，理清楚。卖出去的时候，成本自动从库存里扣。

**适用**：做实物买卖的小老板（小网店、社区团购、摆摊带货、小超市）/ 想知道「我这堆货到底压了多少钱」「卖一单到底赚多少」的场景。

> 这是进阶功能。普通用户默认看不到「商品」「库存」入口——先到 **设置 → 开启商家进阶功能**，左侧才会出现它们。纯记流水、不碰货的，跳过本篇即可。

---

## 一分钟看懂

衡记里关于「货」的事，分两页：

- **商品**：登记你卖什么——名字、进价、售价、单位。这是基础资料，填一次就行。
- **库存**：登记货的进出——进了多少、现在剩多少、值多少钱、盘点对差。

一条贯穿全篇的规则：**所有商品默认都做库存追踪**。你不用勾选「这个商品要不要管库存」，只要它是实物，衡记就替你盯着在手数量和成本。唯一的例外是「纯报价 / 服务」（见下）。

---

## 商品：登记你卖什么

到「商品」页，最下面「新增商品」卡片，填：

| 字段 | 说明 | 必填 |
|---|---|---|
| 名称 | 商品名，不能和已有商品重名 | 是 |
| 单位 | 个 / kg / 箱…… 显示在数量旁边 | 否 |
| 进价 | 你进货的成本价，元 | 否（留空记 0） |
| 售价 | 你卖出的价，元 | 否（留空记 0） |
| 期初库存数量 | 见下文「开账时手里已有货」 | 否 |
| 期初单价 | 同上，留空就用进价 | 否 |

进价、售价都只是**默认值**，开单或进货时还能临时改。填了以后，[开单](orders.md)选这个商品会自动带出售价，进货会自动带出进价，少敲几次键盘。

商品列表里每行点 **编辑** 改资料，点 **归档** 把不卖的商品收起来（归档后开单选不到，但历史记录都在，随时能 **恢复**）。

### 纯报价 / 服务：不进库存的那种

如果你卖的是设计费、打样费、跑腿费这种**没有实物、不占库存**的东西，新增商品时勾上 **「纯报价 / 服务」**。勾了之后：

- 这个商品不做库存追踪，「库存」页里不会出现它；
- 卖它不结转成本（它本来也没有进货成本）；
- 「期初库存」那两个框会自动收起——纯报价没有库存可言。

其余实物商品，**不用管这个勾**，默认就是库存追踪。

### 开账时手里已有货（期初库存）

你刚开始用衡记时，仓库里多半已经堆着一批货。新增商品时填 **期初库存数量**（和可选的 **期初单价**），衡记就把这批存量直接计进库存，不影响你的现金账户——因为这货不是今天花钱进的，是开账前就有的存量。

> 期初单价留空时，用进价。两者都为 0（不知道值多少）时，只记数量、不记金额。

## 库存：货的进出与盘点

到「库存」页，最上面一行就是 **库存值合计**（你手里所有货按成本算，值多少钱，人民币计）。

下面每个商品一行，显示三个关键数：

- **在手 N**：现在还剩多少（卖一单自动减、进一次货自动加）；
- **均价**：每件的平均成本（见下文「原理」）；
- **值**：这个商品的库存总值 = 在手 × 均价。

在手数若为 0 或负数（超卖），数字会标红提醒你。

### 进货入库（现结 / 赊账）

「进货 / 补库存」卡片，选商品、填数量和进价，再选 **付款方式**：

1. **现结**——当场付钱。选一个**付款账户**（你的现金、支付宝、银行卡等人民币账户），钱从那个账户扣，货进「库存商品」。
2. **赊账（记应付）**——先欠着。选一个**供应商**（先去[供应商](purchases.md)页加好），货进库存，欠款记到这个供应商名下。日后到[供应商](purchases.md)页还款。

填好点 **进货入库**。在手数量和库存值立刻更新。

> 库存按人民币本位，所以进货付款账户只能选**人民币（CNY）资产账户**——外币账户、应收账款这类不会出现在下拉里。

### 不够卖怎么办：自动生成草稿采购单

这是统一库存模型最省心的地方。[开单](orders.md)时如果某个商品**在手不够本单要的量**，衡记不会拦着你，而是：

1. 把这张订单标成 **「待采购」**；
2. 自动给每个缺货商品生成一张 **草稿采购单**（数量正好等于缺口，单价预填该商品进价）。

你只要到[采购](purchases.md)页确认这些草稿（补上供应商、采购价），货补上、订单就能转「待发货」。换句话说：**缺货不影响你先把单开了**，补货流程衡记替你起好头。

### 盘点 / 库存调整（盘盈盘亏）

实际数和账上数对不上时——损耗、报废、点错、丢货——用盘点把它们对平。在某商品那行点 **盘点 / 调整**：

1. **实际数量**：填你盘点点出来的真实在手数；
2. **原因**（必填）：盘点对差 / 报废 / 损耗……记进流水备查；
3. **日期**；
4. 如果实际数 **大于** 在手（盘盈），会多出一个 **盘盈入账单价** 框——留空就按当前均价入账。

点 **确认调整**：

- **盘亏**（实际 < 在手）：少掉的货按当前均价算损失，计入 **「库存损溢」**；
- **盘盈**（实际 > 在手）：多出的货按你填的单价（或当前均价）入账，同样进 **「库存损溢」**。

盘点不动你的现金账户——它只是把账面库存调到和实际一致，差额记成一笔损溢。

---

## 原理（点到为止）

你只管「记一笔」，下面这些复式分录是衡记**自动**生成的，看不懂可以跳过：

- **进货**（现结）：付款账户的钱 → 转进「库存商品」（资产换资产，总钱没变，只是从现金变成了货）；赊账则记一笔欠供应商的「应付账款」。
- **卖出**：[订单](orders.md)完成时，按 **移动加权平均成本** 把卖掉那部分货的成本从「库存商品」结转到「营业成本」——这就是你这单真实赚了多少的关键。
- **移动加权平均成本**：每进一批新货，价格不一样，衡记就把新旧货的总成本除以总数量，得到一个新的「平均每件成本」。卖货时就按这个均价扣成本。在手清零时均价归 0，不把零头误差带进下一批。
- **盘盈盘亏 / 期初库存**：分别用「库存损溢」「期初余额」科目做对方，保证账永远是平的。

这些科目（库存商品、营业成本、库存损溢、期初余额、应付账款……）都是**第一次用到时自动建**的，你不用手动去[账户](accounts.md)页创建。
