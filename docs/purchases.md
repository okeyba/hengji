# 采购 · 供应商 · 应付

进货补货、付货款、记一笔运费——衡记把这些都收在「采购」和「供应商」两页。先现结、先赊账都行，欠了谁、什么时候该还，它替你盯着。

**适用**：要进货 / 找供应商赊账的小老板（小网店、社区团购、小餐馆备料）/ 场景＝既要补库存，也要记快递费、办公用品这类零碎采购。

> 采购、供应商这些是「商家进阶功能」。普通用户默认看不到——到 **设置 → 开启商家进阶功能** 打开后，菜单里才会出现「采购」「供应商」。极简记账不需要碰这一篇。

---

## 一、采购页：一笔进货去哪了

打开「采购」，上半屏是所有采购单的清单，下半屏是「新建采购单」。每张采购单都带两个标签：**去向**（这笔钱买的东西算什么）和 **付款方式**（现结还是赊账）。

去向只有两种，对应两条完全不同的账：

| 去向 | 含义 | 例子 | 钱进了哪 |
| --- | --- | --- | --- |
| **入库存** | 进货补库存，按数量 × 进价入「库存商品」 | 进 100 件 T 恤备着卖 | 进库存的移动加权均价池，卖出时才算成本 |
| **费用** | 直接计入某个费用科目，不进库存 | 快递费、办公用品、打印纸 | 当场就是费用，不挂库存 |

> 还有第三种叫「**代采**」——为某张客户订单临时去采购。它不在这页新建，而是在 **[订单](orders.md)** 页里那张单上点「为此单采购」生成。下面第四节专门讲。

### 新建一张「入库存」采购

1. 去向选 **入库存**。
2. 选付款方式（现结 / 赊账，见下一节）。
3. 选 **商品**（没有就先去 [商品](inventory.md) 页加）。
4. 填 **数量** 和 **进价（元/单位）**。进价留空会自动用商品上记的成本价。
5. 选日期，点「新建采购」。

这批货就进了库存，在手数量和均价随之更新。卖出去时系统再按均价结转成本——你这一步只管「进了多少、花了多少」。

### 新建一张「费用」采购

1. 去向选 **费用**。
2. 选 **费用科目**（运费、办公用品之类的支出科目；没有就先去 [账户](accounts.md) 页加一个）。
3. 填 **金额（元）** 和 **说明**（如「快递费」「办公用品」）。
4. 选日期、付款方式，点「新建采购」。

这笔就直接进了那个费用科目，不碰库存。适合所有「买了就花掉」的零碎开销。

## 二、现结 vs 赊账：当场付清，还是先欠着

每张采购单都要选付款方式：

- **现结**：当场从某个 **付款账户** 付清（微信、支付宝、对公账户……）。要选一个人民币账户，钱直接出账。
- **赊账（记应付）**：货先拿、钱后付，欠的记到对应 **供应商** 名下的应付账款。这就是「应付」的来源。

> 选赊账前得先有供应商。没有的话，付款方式那栏会提示你先去「供应商」页加一个。
>
> 采购按 **人民币本位**：现结的付款账户、赊账后的还款，都得是人民币（CNY）账户。

清单里每张单右侧都标着「现结 / 赊账」和金额，一眼能分清哪些已经付清、哪些还欠着。

## 三、供应商档案与账期

到「供应商」页，最下面「新增供应商」可以建档：

1. **名称**（必填，不能和已有供应商重名）。
2. **电话**（可选）。
3. **默认账期（天）**：填 `0` 就是现款现货；填 `30` 表示这家给你 30 天账期。可选，留空当 0 处理。

账期是用来算「什么时候该还」的——从采购那天起加上账期天数，就是应还日。它只是提醒线，不会自动扣款。

每个供应商一行，显示电话、账期、以及当前还欠它多少（「应付 ¥…」或「已结清」）。每行右边有：

- **还款**（还欠钱时才出现）——见下面第五节。
- **改名**——改了名，它名下的应付账款会跟着改，不会把欠款拆成两笔。
- **归档**——不再合作的供应商收起来。还欠着钱也能归档（会提醒你），历史账保留，随时可「恢复」。

## 四、代采：为某张订单临时采购

如果你是「接到单才去进货」的模式（社区团购、代购很常见），流程不在采购页，而在订单页：

1. 在 [订单](orders.md) 页开单时，如果某个商品库存不够，订单会自动转「待采购」并生成一张草稿采购单。
2. 在那张订单上点 **为此单采购**，选供应商、填每个采购品的采购价，选现结 / 赊账，点 **确认采购**。
3. 确认后订单转「待发货」。这笔采购成本直接挂在这张订单上，等订单完成时一并结转成本——不进库存均价池，不和你平时备货的库存混在一起。

> 如果开单后发现库存其实够了，也可以在那张草稿采购单上点「作废」，订单照样能从库存发货。

代采的单子会出现在采购页清单里，标签是「代采」。但你不能在采购页**新建**代采——它只能从订单生成，因为它得知道是为哪张单买的。

## 五、应付账款：你欠供应商多少

只要有一笔赊账采购，就产生了应付。衡记按 **每个供应商分开记**（一个供应商一个应付子科目），和 [应收账款](orders.md)（按客户分开记）正好是一对镜像：应收是别人欠你，应付是你欠别人。

### 应付概览与账龄

「供应商」页顶部，只要有欠款就会出现 **应付概览**：

- **应付** 合计：你总共还欠供应商多少。
- **预付**（如有）：你多付出去、供应商还欠你的。
- **账龄分桶**：把每笔赊欠按「欠了多久」（从采购日算起）分成四档——

| 账龄 | 提示色 |
| --- | --- |
| 0–30 天 | 正常 |
| 31–60 天 | 正常 |
| 61–90 天 | 偏黄（该留意） |
| 90 天以上 | 标红（拖太久了） |

越往后越显眼，方便你优先安排把欠得久的先还掉。

### 还款

1. 在「供应商」页找到要还款的供应商，点它那行的 **还款**。
2. 展开的小表单里：**付款金额** 已预填「当前还欠的全部金额」（也可以改成部分还）、选日期、选 **付款账户**（人民币账户）。
3. 点 **确认付款**。

还款后，这家的应付相应减少，结清了就显示「已结清」。还款按「先欠先还」（FIFO）顺着抵——你不用挑还的是哪一笔，系统自动从最早那笔欠款开始冲。

### 到期提醒

给供应商设了账期（>0）后，当某笔应付临近到期或已逾期，会在账本顶部提醒你去安排付款。没设账期或全部结清就不打扰。这条提醒和 [应收](orders.md) 的收款提醒共用一处，让你一眼看见「这周要收的钱 / 要付的钱」。

## 原理：背后那两笔分录（了解即可）

你只是「记一笔进货」，复式分录是衡记自动配平的，平时不用管。展开看一眼，心里更有底：

- **现结进货**：库存商品（增）← 付款账户（减）。钱变成了货。
- **赊账进货**：库存商品（增）← 应付账款 / 某供应商（欠款增）。货先拿，欠记上。
- **费用采购**：某费用科目（增）← 付款账户 或 应付账款（减 / 欠款增）。
- **还款**：应付账款 / 某供应商（欠款减）← 付款账户（减）。把欠的结掉。

每一笔都是借贷相等、自动平衡——你看到的「还欠多少、账龄几天」，都是从这些分录实时算出来的，不是手填的死数。
